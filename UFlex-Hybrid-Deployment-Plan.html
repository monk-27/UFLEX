Section 02 — Migration Plan

3-Phase Rollout Plan

We move in three clear phases. Each phase is independent — we can pause after any phase and the site keeps working.

1

Deploy Frontend to Vercel

⏱ Phase 1 — Do This Now (1-2 Days)
  • Connect GitHub repo to Vercel — one-time setup
  • Set environment variable: NEXT_PUBLIC_API_BASE_URL=https://yourdomain.com
  • Update all fetch() calls in Next.js to use full cPanel domain URL
  • Add CORS headers to cPanel PHP files (allow Vercel domain)
  • Point domain DNS to Vercel — website goes live on Vercel
  • cPanel PHP + MySQL continues running exactly as before
🟢 Ready to Execute Now
2

Migrate PHP APIs to Next.js (Optional)

⏱ Phase 2 — When Ready (2-4 Weeks)
  • Rewrite submit-contact.php → Next.js API route (app/api/contact/route.ts)
  • Rewrite submit-quote.php → app/api/quote/route.ts
  • Rewrite get-unclaimed-dividend.php → app/api/dividend/route.ts
  • Replace PHPMailer with Nodemailer (same SMTP config)
  • Move file uploads to Vercel Blob or Cloudinary
  • Migrate MySQL to PlanetScale / Railway / Supabase
🟣 Planned — Not Urgent
3

Retire cPanel Completely

⏱ Phase 3 — Future (When Phase 2 is Stable)
  • Run both systems in parallel for 2-4 weeks to verify stability
  • Confirm all email alerts and DB writes working on new system
  • Keep cPanel emails (SMTP) if needed — or move to Resend/SendGrid
  • Cancel cPanel hosting subscription
  • 100% on Vercel + Cloud DB — fully modern stack
🌱 Future Goal

Section 04 — Environment Variables

Managing .env Files in Hybrid Setup

Your secret keys and credentials are managed separately on each platform. Nothing needs to move — just one new variable to add on Vercel.

▲ Vercel Dashboard

Only Next.js frontend variables go here. Set in Vercel → Project → Settings → Environment Variables.

NEXT_PUBLIC_API_BASE_URL
= https://yourdomain.com (your cPanel domain)
NEXT_PUBLIC_SITE_URL
= https://your-app.vercel.app

⚠️ Never put DB passwords or SMTP passwords here

🖥️ cPanel .env.local

Already exists. Keep it exactly as-is. PHP reads it directly. No changes needed.

DB_HOST
DB_USER
DB_PASSWORD_B64
DB_NAME
SMTP_HOST / SMTP_USER
SMTP_PASSWORD / SMTP_PORT

✅ Leave untouched — PHP reads these automatically

Section 05 — One Required Change

The Only Change Needed on cPanel

Just add 3 lines to your _security-helper.php file on cPanel to allow Vercel to call your PHP APIs.

⚡ Why This Is Needed

Browsers have a security rule called CORS (Cross-Origin Resource Sharing). When your Next.js app on Vercel tries to call your PHP on cPanel, the browser blocks it unless cPanel explicitly says "yes, Vercel is allowed." These 3 lines do exactly that.


// Add these lines at the very TOP of _security-helper.php

// Allow Vercel frontend to call this PHP API
header('Access-Control-Allow-Origin: https://your-app.vercel.app');
header('Access-Control-Allow-Methods: POST, GET, OPTIONS');
header('Access-Control-Allow-Headers: Content-Type');

// Handle preflight request (browser sends this before POST)
if ($_SERVER['REQUEST_METHOD'] === 'OPTIONS') {
  http_response_code(200);
  exit;
}
